Good. These are four distinct feature layers so let me break down what each one actually requires from the codebase.
Photos for all wrestlers — Right now you’ve got playerPhotoKey on some chapters but nothing on the wrestler objects themselves. You’ll need an image field on the Wrestler interface and then asset files for 40+ characters. The question is whether you’re doing original art, pulling from real event photography you own, or generating assets through your Manus pipeline. Given your existing asset library workflow that’s probably the cleanest path. Each wrestler needs at minimum a portrait and ideally a ring entrance shot if you want visual variety between cutscenes and match screens.
Tag match mechanics — This is the biggest structural change. Right now your CareerChapter has a single opponentId which forces everything into singles logic even when the historical result is a 6-man tag elimination. You need:
A matchType field on the chapter — singles, tag, 6-man, elimination, rumble, etc. A teammates array and an opponents array instead of a single opponentId. Then the match engine needs to handle partner AI, hot tags, legal man tracking, and team-based win conditions. The elimination matches like Retribution need sequential pins with stamina carrying over. The Riot Rumble needs its own over-the-top-rope system with your bottom-rope exploit as a mechanic the player can actually attempt.
Stipulation-specific mechanics — This is the variety layer. Each stipulation type needs its own win condition and available action set. Hardcore matches add a weapons inventory — chairs, tables, kendo sticks — with environmental interaction replacing the standard moveset. Ladder matches need a climb meter that opponents can interrupt. Cage matches need escape-or-pin dual win conditions. The Casket match from Era 1 needs its own push-to-casket mechanic. Your stipulation field already exists on every chapter, so the data is there — you just need the engine to read it and swap rulesets.
Legacy Mode vs Career Mode split — This is actually the cleanest change architecturally. Legacy Mode is what you have now — locked chapters, historical results as the benchmark, forced replay on deviation. Career Mode would use the same wrestler database and match engine but with a branching path system — you pick your opponents, choose which factions to build or burn, and the storyline reacts. Career Mode is where you’d want a reputation system, a heat meter that tracks crowd reaction across shows, and booking decisions that have consequences three events later. That’s a fundamentally different game loop but it shares all the same assets and combat mechanics.
The question is priority order. What are you building next — the tag system, the stipulation engine, the photos, or the mode split?​​​​​​​​​​​​​​​​
